Security Considerations

For the Company:-

1. Admin page should not be accessible to all users:

- Only selected administrators should be able to access the admin page. This is to avoid and prevent potential breach into the system database from external attackers. This could be done using a username and password login to grant necessary permissions to make changes to the database.

2. Avoid storing plaintext passwords in databases:

- If the database on the system is compromised, all the passwords are immediately visible and can easily be associated with its respective users.
- Internal malfeasors (people who work for the organization) can abuse these passwords by using them to access personal user information such as credit card or bank account details.
- Instead, store salted hashed passwords. (Salted hashed passwords: Before going through the password hashing function, random characters are added to the password which gives out a different output every time, making it more difficult for hackers to crack it.)

For the Users:-

1. Avoid reusing same/similar passwords:

- If the user has been compromised on one system, the attacker might use the same password on other platforms, resulting in multiple account breaches.
